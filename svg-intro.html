Do Not be afraid of SVG

Utilize It On Your Web Today

by Marek Raida / @xrevelon

Scalable Vector Graphics

  • W3C approved standard since 1999
  • focused on 2D vector space
  • vectors = super quality for every screen resolution
  • HTML5, CSS and JS friendly
  • XML based markup (even when part of HTML5 document!)
  • DOM based with events well supported
  • well supported, even in MSIE 9+ (easy fallbacks possible)
  • accessible (role attribute, title/desc tags)

How to create SVG

  • manually (just like you do HTML, CSS)
  • profi editor like Illustrator or Inkscape
  • online editor like SVG-Edit
  • download and utilize documents from internet
  • generate it via scripting

SVG Basics

  • shape primitives: path, line, rect, circle, ellipse, polyline, polygon…
  • organization: defs, g, svg, use, symbol, a, image, foreignObject…
  • advanced: clipPath, mask, marker, pattern, gradients, filters…
  • texts: text, tspan, textPath…
  • animations: set, animate, animateTransform, animateMotion

viewPort vs. viewBox

SVG canvas is infinite. ViewPort is defined by width and height and is for internal percentage sizing use + it gives to document's parent (HTML, SVG) its dimensions

  <svg width="200" height="200"><!-- content --></svg>

ViewBox is defined by viewBox and defines document's canvas visibility box. Use it a lot!! Four values stand for definition of: left top width height

  <svg viewBox="-50 -50 200 200"><!-- content --></svg>

No one of them is mandatory, but they are useful and they can co-exists easily, if you want to

  <svg viewBox="-50 -50 200 200" width="100" height="100"><!-- … --></svg>

Integration ways

There are many ways how to integrate SVG in your web project. I recommend to stick to these ones:

  1. inline svg markup
  2. object tag
  3. image (img) tag
  4. CSS background property

Inline SVG markup

Mind necessity to keep XML syntax for all SVG markup!

  …
  <h2>Some Title</h2>
  
  <svg width="50" height="50" xmlns="http://www.w3.org/2000/svg">
    <circle cx="25" cy="25" r="20" fill="orange"/>
    <line x1="0" y1="0" x2="50" y2="50" stroke="green" stroke-width="3"/>
  </svg>
  <script>
    if (document.implementation.hasFeature(
          "http://www.w3.org/TR/SVG11/feature#Shape", "1.1")) {
      // optional fallback here
    }
  </script>
  <p>Text of some paragraph</p>
  …

Advantages

  • applicable CSS from embedding document (also currentColor support)
  • manipulable via JavaScript
  • no extra file to load
  • accessible

Disadvantages

  • not cacheable
  • cross-browser (MSIE) fluid size trick needed

Use if you need stretching, fixed sizes are OK without it.

Image tag integration

Good old, familiar and easy way to go…


    <img width="323" height="153" src="svg/turtle.svg" alt="Turtle"
        onerror="this.onerror=null; this.src='fallback.png'"/>
    <!-- with possible fallback using onerror attribute -->

Advantages

  • cacheable, as it is external file
  • intensive pre-compress possible
  • safe and simple (sandboxed)
  • SMIL is allowed (animated GIF substitute, except for MSIE)

Disadvantages

  • external (linked) CSS stylesheet not applicable
  • owner document's CSS stylesheet not applicable to individual SVG elements
  • cannot be manipulated via JavaScript (not even internally)
  • interactivity disabled (links, DOM events)

Fluid design MSIE trick

Similar to object tag - you need to set width. Live demo


    <style>
      img {
          width: 100%;
      }
    </style>
    
    <img width="323" height="153" src="svg/turtle.svg" alt="Turtle"/>

Understanding path

Can replace every other shape. Some available commands:

  • M = Move To X,Y
  • L = Line To X,Y
  • C = Cubic Bézier To x1,y1 x2,y2 X,Y
  • Z = Close Path (to the start)

Use reflections

use tag creates live reflection of any existing node. It not only saves lines of codes, but can do true miracles!

  <g id="complexElementID" class="…">
    <circle …/> <rect …/> <text …/> <path id="somePathID" …/>
  </g>
  <use xlink:href="#complexElementID" transform="scale(2)" class="c1 …"/>
  <g class="whatever">
    <use xlink:href="#complexElementID" transform="scale(5)" class="c2 …"/>
    <use xlink:href="#somePathID" transform="rotate(180)" class="c3 …"/>
  </g>

References are using xlink:href="#refID" syntax.

Limitations

MSIE does not support CSS transformations to SVG content, so you have to stick to direct tag properties

MSIE also does not support CSS Transitions and Animations of otherwise supported SVG related CSS properties, which leaves us with reasonable utilization of only: opacity.

It is also the only browser not supporting SMIL animation model, so with SVG you're out of luck.

Scripting could solve all these issues, but it's a pity.
