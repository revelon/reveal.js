Do Not be afraid of SVG

Utilize It On Your Web Today

by Marek Raida / @xrevelon

Scalable Vector Graphics

  • W3C approved standard since 1999
  • XML based markup, DOM manipulation
  • focused on 2D vector space
  • vectors = super quality for every screen resolution
  • HTML5, CSS and JS friendly
  • well supported, even in MSIE 9+ (easy fallbacks possible)

How to create SVG

  • manually (just like you do HTML, CSS)
  • profi editor like Illustrator or Inkscape
  • online editor like SVG-Edit
  • download and use files from internet
  • generate via scripting

SVG Basics

  • primitives: line, rect, circle, ellipse, polyline, polygon, path
  • organization: defs, g, svg, use, symbol, a, image, foreignObject...
  • advanced: clipPath, mask, marker, pattern, radial/linearGradient, mask, filters...
  • texts: text, tspan, textPath, tref
  • animations: set, animate, animateTransform, animateMotion

Recommended integration ways

There are many ways how to integrate SVG in your web project. I recommend to use these ones:

  1. inline markup
  2. object
  3. image
  4. CSS background

Inline markup


    ...
    <h2>Some Title</h2>
    
    <svg width="50" height="50" xmlns="http://www.w3.org/2000/svg">
      <circle cx="25" cy="25" r="20" fill="green"/>
      <line x1="0" y1="0" x2="50" y2="50" stroke="red"/>
    </svg>
    <script>
      if (document.implementation.hasFeature("http://www.w3.org/TR/SVG11/feature#Shape", "1.1")) {
        // optional fallback here
      }
    </script>
    <p>Text of some paragraph</p>
    ...

Advantages

  • applicable CSS from embedding document (also currentColor support)
  • manipulable via JavaScript
  • accessible

Disadvantages

  • not cacheable
  • cross-browser (MSIE) fluid size trick needed

Fluid design MSIE trick

if needed only, fixed sizes are OK... Live demo link 
    <style>
      .wrapper {
          width: 50%;
          height: 0;
          padding-top: 50%; /* (svg-height / svg-width) * wrapper-width */
          position: relative;
      }
      svg {
          position: absolute;
          top: 0;
          left: 0;
      }
    </style>
    <div class="wrapper">  
      <svg viewBox="0 0 50 50"></svg>
    </div>

Object tag integration

    </object>
    <p>Text of some paragraph</p>

Advantages

  • applicable CSS from external stylesheet
  • manipulable via JavaScript (both externally and internally)
  • cacheable, as it is external file
  • intensive pre-compress possible
  • accessible ??

  • cross-browser (MSIE) fluid size trick needed

Image tag integration

Good old way...


    <img width="178" height="238" src="svg/hourglass.svg" alt="Hourglass"
      onerror="this.onerror=null; this.src='fallback.png'"/>

Advantages

  • cacheable, as it is external file
  • intensive pre-compress possible
  • safe and simple

Disadvantages

  • external CSS stylesheet not applicable
  • owner document CSS stylesheet not applicable (only globally)
  • cannot be manipulated via JavaScript (not even internally)

Fluid design MSIE trick

if needed only... Live demo link 

    <style>
      img {
          width: 100%;
      }
    </style>
    
    <img width="178" height="238" src="svg/hourglass.svg" alt="Hourglass"/>

CSS background image


    <style>
      .holder {
        width: 50%;
        height: 50%;
        background: url('img/hourglass.png'); /* possible fallback trick */
        background-image: url(svg/hourglass.svg), none;
      }
    </style>
    
    <div class="holder"/>
 

Advantages

  • cacheable, as it is external file
  • intensive pre-compress possible
  • no tricks needed for modern browsers

Disadvantages

  • external CSS stylesheet not applicable
  • owner document CSS stylesheet not applicable (only globally)
  • cannot be manipulated via JavaScript (not even internally)
